
# Oppdatering v4.6.2 - Pre-Rotation Split Logic

Denne oppdateringen løser problemene der A3-oppslag ble splittet feil fordi bildefilen var rotert 90 grader (sidestilt skanning).

### Endringer:
*   **Rotate-then-Split**: Systemet utfører nå en fysisk rotasjon på bildefilen *før* den splitter bildet i to A4-sider. Dette sikrer at 50/50-kuttet alltid treffer midten av arket vertikalt, slik det skal.
*   **Canvas-basert Prosessering**: Bruker Canvas API til å utføre rotasjon og splitting i ett koordinert steg, noe som forhindrer bildeforvrengning.
*   **Omnidireksjonell Deteksjon**: KI-en rapporterer rotasjon for hele arket, og systemet bruker denne informasjonen til å "rette opp" bildet før splitting skjer.
*   **Bevaring av Oppløsning**: Optimalisert lagring av de splittede bildene for å bevare maksimal lesbarhet av håndskrift.
*   **Stabilisering av Metadata**: Siden bildene nå roteres før lagring, fjernes behovet for dynamisk rotasjon i CSS i kontroll-steget for disse sidene, noe som gir en raskere og mer stabil brukeropplevelse.
