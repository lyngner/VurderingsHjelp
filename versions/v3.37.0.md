
# Oppdatering v3.37.0 - Hierarkisk Rettemanual

Denne oppdateringen innfører en mer logisk og profesjonell struktur på rettemanualen ved å skille mellom oppgave og deloppgave.

### Endringer:
*   **3-nivå Hierarki**: Innført `taskNumber` (f.eks. "1") og `subTask` (f.eks. "a") som egne felt i databasen og KI-svarene.
*   **Forbedret Navigasjon**: Sidebar i rettemanualen bruker nå de eksplisitte feltene for å gruppere oppgaver, noe som eliminerer feil som tidligere oppstod ved regex-basert gjetting.
*   **Ryddigere Hovedvisning**: Deloppgaver vises nå tydeligere som barn av sin respektive hovedoppgave.
*   **Strammere KI-kontrakt**: Oppdatert `responseSchema` i `geminiService.ts` for å tvinge modellen til å levere data i det nye formatet.

### Teknisk Gjennomføring (Hvordan):
*   **Schema-utvidelse**: Oppdatert `types.ts` med de nye feltene.
*   **Prompt-Engineering**: Instruert Gemini 3 Pro til å alltid splitte oppgave-IDer (som "1a") i sine bestanddeler.
*   **UI-Refactoring**: Skrevet om grupperingslogikken i `RubricStep.tsx` for å utnytte de nye metadata-feltene direkte.
