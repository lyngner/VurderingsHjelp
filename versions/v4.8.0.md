# Oppdatering v4.8.0 - Thinking Architecture & Physical Rotation

Denne oppdateringen løser problemene med upålitelig layout-deteksjon og skeive bilder ved å innføre en dypere analyse-pipeline.

### Endringer:
*   **Gemini 3 Pro Thinking Config**: Alle OCR- og transkripsjonskall bruker nå Gemini 3 Pro med en dedikert `thinkingBudget`. Dette gir modellen tid til å resonnere grundig over sideoppsett, rotasjon og oppgave-struktur før den returnerer JSON.
*   **Fysisk Rotasjons-Pipeline**: Bilder blir nå fysisk rotert via Canvas API *før* de lagres i databasen hvis Gemini detekterer feil orientering. Dette fjerner behovet for CSS-transformasjoner og sikrer at alle sider vises som perfekte portretter i Kontroll-modulen.
*   **Aggressiv A3-Splitting**: Forbedret logikk for å skille mellom en rotert A4 og et ekte A3-oppslag. Gemini er nå betydelig mer treffsikker på å splitte ark som inneholder to kolonner med tekst.
*   **Intelligent Oppgavematching**: Oppdatert logikken som sammenligner funne oppgaver med rettemanualen for å være mer fleksibel (f.eks. automatisk splitting av "1A" til "1" + "A" hvis det er slik manualen er strukturert).
*   **Digital Presisjon**: Bedre instruksjoner for Word-dokumenter for å fange opp romertall og komplekse lister.
