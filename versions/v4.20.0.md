
# Oppdatering v4.20.0 - Physical First A3 Pipeline

Denne oppdateringen løser de gjentagende problemene med A3-oppslag som ikke ble splittet eller rotert korrekt.

### Endringer:
*   **Tvungen Dobbel-Retur**: Gemini 3 Pro er nå eksplisitt instruert til å returnere TO separate objekter i JSON-listen dersom et A3-oppslag detekteres. Dette garanterer at begge sider i spredningen blir prosessert.
*   **Fysisk Rotasjon-Først**: Alle bilder blir nå fysisk rotert og lagret på nytt i databasen basert på KI-ens detekterte rotasjon. Dette fjerner avhengigheten av upålitelig CSS-transformasjon i visningen.
*   **Høyoppløselig Lagring**: Oppdaterte `splitA3Spread` til å returnere 95% kvalitet JPG for å bevare alle detaljer i elevens håndskrift etter splitting.
*   **Regresjonsvern**: Oppdatert Master README med krav om "Physical Rotation Pipeline" for å hindre at systemet faller tilbake på rent metadata-basert rotasjon.
*   **Renere Visning**: Siden bildene nå lagres "rett vei", er visningen i Kontroll-modulen mer stabil og raskere.

### Teknisk:
*   Optimalisert `useProjectProcessor.ts` til å håndtere parallelle bildeoperasjoner via Canvas API.
*   Sikret at `rotation` i metadata settes til 0 etter fysisk prosessering for å unngå dobbelt-rotasjon.
