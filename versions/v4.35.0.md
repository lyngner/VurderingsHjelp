
# Oppdatering v4.35.0 - Deep XML Header Extraction

Denne oppdateringen løser det kritiske problemet der Kandidatnummer i Word-dokumenter (.docx) ble oversett fordi de lå i dokumentets "Headers" (topptekst).

### Endringer:
*   **Deep XML Extraction**: Systemet bruker nå `jszip` til å pakke ut Word-filen og lese tekst direkte fra `word/header*.xml`. Dette fanger opp data som Mammoth og andre biblioteker normalt ignorerer.
*   **Priority Prepend**: Topptekst-data legges nå øverst i transkripsjonen i en egen seksjon merket `[HEADER-DATA]`. Dette fungerer som et signal til KI-en om at dette er den primære kilden til Kandidat-ID.
*   **Forsterket ID-Anker**: Gemini 3 Pro er nå instruert til å prioritere data i `[HEADER-DATA]`-seksjonen ved fastsettelse av `candidateId`.
*   **Robust Word Parsing**: Bedre feilhåndtering ved lesing av komplekse Word-strukturer.

### Teknisk:
*   Implementert `extractWordHeaders` i `fileService.ts`.
*   Oppdatert `analyzeTextContent` i `geminiService.ts` med nye regler for header-prioritering.
*   Lagt til krav om "Deep XML Header Extraction" i `TECHNICAL_STANDARD.md`.
