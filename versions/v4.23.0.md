
# Oppdatering v4.23.0 - Atomic Stability & A3 Logic Fix

Denne oppdateringen løser problemene med "forsvunne kandidater" og gjør A3-splittingen enda mer pålitelig.

### Endringer:
*   **Atomic Updates**: Skrevet om `integratePageResults` til å bruke funksjonelle React-oppdateringer. Dette eliminerer race-conditions som tidligere førte til at kandidatlisten ble tømt ved rask prosessering.
*   **Status-synkronisering**: Fil-listen i Setup-steget viser nå korrekt 'Analyserer' status mens bakgrunnsarbeidet pågår. Dette gir læreren trygghet om at systemet jobber.
*   **Strammere A3-kontrakt**: Oppdatert Gemini-instruksen til å være diktatorisk på at landskapsskanninger SKAL returnere to objekter hvis de inneholder to spalter. Dette fjerner "lathet" fra AI-modellen.
*   **ID-Sikring**: Kandidat-IDer blir nå konsekvent renset for tekststøy (f.eks. "Kandidat 101" -> "101") før de lagres, noe som sikrer korrekt gruppering.
*   **Regresjonsvern**: Oppdatert `TECHNICAL_STANDARD.md` med krav om Atomic Updates for asynkron state-håndtering.

### Teknisk:
*   Fikset bug i `useProjectProcessor.ts` der rubric kunne være stale i asynkrone callbacks.
*   Sikret at `processingCount` og progress-baren er perfekt synkronisert med fil-listen.
